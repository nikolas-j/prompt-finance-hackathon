How to improve this system next

This page summarizes why the confidence approach is useful today and where it can deliver even more value with the next development steps.

1) Fast experimentation across retrieval pipelines

The architecture is intentionally modular: we can swap chunking, retrieval, and judge settings without rewriting the full app. That means we can run the same question bank automatically on each variant and compare outcomes quickly.

Practical impact: we can test ideas rapidly, keep what improves judged correctness, and discard what does not.

2) Higher-value signals from advanced traversal

Graph and hybrid traversal already improve the retrieval pool. More advanced traversal systems can provide richer confidence signals, such as multi-hop consistency, cross-source agreement, path stability, and repeated statute confirmation across independent retrieval paths.

Practical impact: better retrieval traces produce better confidence estimates.

3) Scale test data beyond 83 questions

Current confidence models are trained with 5-fold cross-validation on 83 questions. This is a strong prototype, but sample size is still small for robust calibration.

With a verified set of around 1,000 questions, we expect materially better generalization, tighter confidence calibration, and more stable feature importance across retraining runs.

4) Why this works especially well in Finnish tax

Finnish tax guidance is a constrained domain: terminology, thresholds, percentages, and legal references are structured and repeated. That makes retrieval quality highly measurable and confidence signals highly informative.

In this type of domain, confidence prediction from retrieval behavior is more reliable than in broad open-domain question answering.

5) Use confidence inside iterative retrieval loops

Confidence should not only be displayed; it should control behavior. If the first answer has low confidence, the system can re-search with alternative retrieval strategies, run focused follow-up queries, or narrow the question scope to reduce uncertainty.

This is directly compatible with GraphRAG and other multi-step agentic retrieval systems: confidence acts as a gating signal for another retrieval pass instead of returning a weak answer as final.
